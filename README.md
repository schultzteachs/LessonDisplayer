Classroom Lesson Displayer
The Problem
When students are absent, the immediate bottleneck is information retrieval. "What did we miss?" is a repetitive query that drains instructional time and creates friction for the student. The classroom lacked a Single Source of Truth (SSOT) for curriculum tracking.

The Solution
This web portal acts as a centralized dashboard displaying weekly lessons for my AP CSP and C# classes. It eliminates administrative overhead by routing absent students directly to the materials they need, ensuring continuity of learning with zero teacher intervention required.

Design Philosophy: High availability and zero barrier to entry for the end-user (the student).

Tech Stack & Approach
Frontend: CSS Tailwind, JavaScript, Google Sheets

Rapid Prototyping: I leveraged Google Gemini to quickly scaffold the initial boilerplate and UI layout. In this workflow, I treated the AI like a junior developer to speed up the drafting phase.

Architecture & Refactoring: All core logic, system architecture, and final code optimizations were handled by me to ensure the platform actually met the specific operational constraints of my classroom.

The "Why"
As a teacher transitioning into Solutions Architecture, my focus is on identifying systemic inefficiencies and building scalable tools to fix them. This project is a lightweight demonstration of that mindset: find the bottleneck, design the system, leverage modern tooling to build it fast, and deploy a solution that works.
